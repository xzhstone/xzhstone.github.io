从零搭建个人博客：技术栈与设计思路

2026 年 06 月 18 日 技术 设计

这篇博客本身就是用本文介绍的方式搭建的。我想记录一下构建过程中的技术选择和设计思考。

为什么不用框架？

在决定搭建博客时，我面临一个选择：用成熟的博客框架（如 Hugo、Ghost、WordPress）还是自己从零写。最终我选择了后者，原因很简单：

  • 极致的控制力 — 每一行代码都知道在做什么，没有黑盒
  • 极小的体积 — 没有框架依赖，页面加载几乎就是文本传输的时间
  • 写作的核心 — 最少的工具噪音，最直接的内容呈现

当然，这也有代价：没有 Markdown 渲染、没有自动 RSS、没有评论系统。但对于一个以写作为核心的站点来说，这些功能可以用轻量的方式逐个添加。

技术架构

整个站点由三部分组成：

数据层：JSON 元数据

所有文章的元信息统一存放在 posts/index.json 中，包括标题、日期、标签和文件路径。首页通过 JS 异步加载这个 JSON 并渲染列表。这样做的好处是新增文章只需要写 HTML + 更新 JSON，不需要改模板。

展示层：纯 HTML + CSS

每个页面都是独立的 HTML 文件，CSS 通过自定义属性（CSS Variables）实现了完整的明暗主题系统。设计上追求：

  • 干净、留白充足，让阅读本身成为焦点
  • 响应式布局，从手机到大屏都有好的体验
  • 明暗主题自动跟随系统，也支持手动切换

交互层：轻量 JavaScript

JavaScript 做了三件事：

  • 主题切换（可持久化到 localStorage）
  • 移动端菜单
  • 文章列表的标签筛选

总代码量 < 200 行 JS，没有框架、没有构建工具。

CSS 主题系统

颜色方案使用 CSS 自定义属性（:root 变量），通过 @media (prefers-color-scheme: dark) 自动适配系统主题。用户手动切换时，在 <html> 上添加 .dark 或 .light 类来覆盖，并将选择存储到 localStorage。

:root { --bg: #fafafa; --text: #1a1a2e; }
@media (prefers-color-scheme: dark) {
  :root { --bg: #0f0f1a; --text: #e2e8f0; }
}
:root.dark { /* 手动暗色覆盖 */ }
:root.light { /* 手动明色覆盖 */ }

部署方案

静态站点的部署方案有很多选择。目前的计划是：

  • GitHub Pages — 简单免费，适合静态站点
  • Vercel / Netlify — 如果需要更高级的功能（如 Serverless 函数），也是一个好选择

静态站点最大的好处就是部署极其简单——直接把文件丢到服务器上就可以访问。

这就是这个博客的搭建过程。如果你也在考虑搭建自己的博客，我的建议是：不管用什么工具，开始写才是最重要的事。
